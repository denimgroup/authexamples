Solutions for different scanners:

  • IBM Rational AppScan
  • HP WebInspect
  • Mavituna Security Netsparker
  • NTObjectives NTOSpider

    IBM Rational AppScan

    This can be handled in AppScan through the use of Custom Parameters. See this blog post for more information. Thanks to Ory Segal (@orysegal) from IBM Rational for sending these details along. Also Ory put up a blog post here with some more information about AppScan's Custom Parameters.

    An alternate approach is to chain AppScan to run through Portswigger BurpSuite. See this blog post for more information on this (somewhat more convoluted) approach.

    HP WebInspect

    The event-based macro UI recorder can be used to handle situations such as these. Also JavaScript can be used to script even more complex behaviors. See this blog post for more information. You can also see the original blog post and PDF from the HP folks.

    Mavituna Security Netsparker

    This can be handled via the scripting facilities in Netsparker. See this blog post for more information. Also, you can download the sample files: XML login script and C# script extension.

    NTObjectives NTOSpider

    This can be handled by using a regex to populate the input name/value pairs. See this blog post for more information.
